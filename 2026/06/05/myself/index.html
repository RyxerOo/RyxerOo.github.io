1. sqlmap 怎么拿shell

使用 --os-shell，前提是数据库有写文件权限、知道网站绝对路径，且 secure_file_priv 允许。sqlmap 会自动写入对应语言的 webshell（如 PHP/ASP/JSP），然后通过它执行命令。

2. MySQL 通过注入点提权方法

  • UDF 提权：将恶意动态库写入插件目录，创建自定义函数执行系统命令。
  • MOF 提权（老版 Windows）：写入 MOF 文件到 system32\wbem\mof，执行任意命令。
  • 写入启动项/计划任务：利用 INTO OUTFILE 写 bat 或 vbs 到启动目录。

3. MySQL 延时注入关键函数，全被过滤怎么办

关键函数：sleep()、benchmark()。
绕过：用 get_lock() 锁等待、笛卡尔积重查询、repeat('a',10000000) 消耗时间，或用 WAITFOR DELAY（SQL Server），或改用 DNS 带外注入替代延时。

4. sqlmap 中 --os-shell 执行的底层原理

先通过注入点执行 SELECT ... INTO OUTFILE/DUMPFILE 将构造的 webshell（如一句话木马）写入网站目录，然后通过该 webshell 传递系统命令并返回结果，形成交互式 shell。

5. 是否会写脚本，都在什么情况下写什么样的脚本

场景：

  • 批量 POC 验证脚本（Python/Go），护网打点或漏洞复现。
  • 资产收集脚本（子域名、端口扫描）。
  • 应急响应中自动化日志分析、IP 封锁脚本。
  • WAF 绕过的 tamper 脚本（sqlmap）。

6. 可疑进程占用 80 端口，查询没有进程号，怎么排查

  • 用 cat /proc/net/tcp 查看原始套接字信息，找出 inode 号。
  • 用 lsof | grep <inode> 尝试定位，或用 fuser -n tcp 80。
  • 可能为 Rootkit 隐藏进程，用 unhide、rkhunter 扫描。
  • 检查加载的内核模块 lsmod | grep -i hide，必要时进救援模式排查。

7. 文件上传

攻击点：前端 JS 校验、MIME 类型、扩展名黑名单（大小写、双写、空字节截断、.phtml 等）、内容检测绕过（图片马 + 包含）、条件竞争、解析漏洞（Apache 多后缀等）。

8. 白名单绕过

利用白名单内文件的功能点，如图片二次渲染保留 webshell 代码、上传 .htaccess 将图片当作 PHP 解析、压缩包解压路径穿越、双文件上传拼接扩展名等。

9. 蚁剑、冰蝎、哥斯拉等攻击流量特征

  • 蚁剑：默认流量 base64 编码，参数名固定（如 @ini_set 特征），无加密。
  • 冰蝎：全加密，协商密钥阶段有固定 AES 加密请求头，响应体加密，心跳包规律。
  • 哥斯拉：支持多种加密器，流量加密但可自定义，特征不明显，需结合行为检测（周期性心跳、固定路径）。

10. XSS 存储型提权方法

劫持管理员 Cookie 登录后台，利用后台功能（模板编辑、文件上传、SQL 执行）写入 webshell 或添加管理员账号，或配合 CSRF 执行敏感操作。

11. SSRF

服务端请求伪造，让服务器发起攻击者指定的请求。利用方式：内网端口探测、文件读取（file://）、攻击内网未授权服务（Redis、MySQL）、云环境元数据窃取。

12. 拿到一个 web 页面你怎么进行渗透？

信息收集（子域名、端口、CMS 指纹）→ 漏洞扫描（已知 CVE、弱口令、注入、上传）→ 手工测试重点功能（登录、搜索、API）→ 利用漏洞获取权限 → 后渗透提权、横向移动。

13. 高危漏洞特点

可直接获取系统权限（RCE、SQL 注入写 shell）、无需认证或仅需低权限、影响核心数据、可造成业务瘫痪、利用简单且稳定。

14. 高危、中危、低危漏洞评分

CVSS v3 标准：

  • 严重（Critical）：9.0 - 10.0
  • 高危（High）：7.0 - 8.9
  • 中危（Medium）：4.0 - 6.9
  • 低危（Low）：0.1 - 3.9
    评分考虑攻击向量、复杂度、权限要求、用户交互、影响范围及数据保密性/完整性/可用性。

15. 渗透、应急的基本思路以及过程中用什么命令

  • 渗透：信息收集（nmap、subfinder）→ 漏洞利用（sqlmap、msf）→ 后渗透（mimikatz、proxychains）。
  • 应急：确认事件 → 隔离 → 排查（netstat -an、ps aux、find 查 webshell、grep 日志）→ 清除 → 恢复 → 复盘。

16. 你在项目过程中有没有遇见什么问题，你是如何解决的

例：某次内网打点时 WAF 拦截 sqlmap，编写 tamper 脚本将 payload 变形为 /*!50000union*/，成功绕过；应急中怀疑受害机存在隐藏进程，通过 unhide proc 找出内核级后门并去除。

17. 如果你挖的中危漏洞被降级，如何说服对方？

展示漏洞链，说明在特定条件下可升级为高危（如信息泄露+弱口令导致系统权限），引用 CVSS 评分逻辑强调影响范围，并给出真实场景模拟，证明其潜在危害。

18. AWD 比赛时队友意见分歧，你怎么做

作为队长，开局分工：一人部署基础防守（改密码、备份、文件监控、waf 脚本），两人快速攻击拿分；中期轮换，防守队员同时分析流量提取新攻击 payload 反打。平衡攻防，不偏废。

19. 领导说“拿到我身份信息又能怎样？”你该如何应对

解释身份信息可被用于精准钓鱼、社工同事、伪造贷款、补全其他数据后入侵内网，给企业带来声誉损失和法律风险，预防成本远低于事后补救。

20. 你对目前安全行业有什么问题？怎么改进

问题：重合规轻实战、产品同质化、人才技能与需求脱节、攻防演练流于形式。改进：推动常态化实战演练、安全左移嵌入开发流程、加强实战型人才培养、促进威胁情报共享和自动化响应。

21. 给你一个公司名称你要怎么进行渗透？

资产收集（子域名、IP、端口、服务）→ 指纹识别（CMS/中间件）→ 漏洞探测与利用（注入、上传、反序列化、弱口令等）→ 拿权限后内网横向、提权、维持。

22. SQL注入的原理和防范

原理：攻击者输入拼接进SQL语句，改变逻辑。防范：参数化查询（预编译），白名单输入校验，最小权限，屏蔽错误，WAF。

23. Linux登录成功的日志存放目录

  • /var/log/secure（CentOS/RHEL）
  • /var/log/auth.log（Debian/Ubuntu）
  • 也可用 last -f /var/log/wtmp 查看。

24. Linux查看进程的命令，怎么判断排查恶意进程？

命令：ps aux、top、pstree、ls -l /proc/PID/exe。排查：看CPU/内存异常进程，启动路径可疑（/tmp、/dev/shm），联网行为（netstat -anp），对比已知正常进程，查隐藏（unhide）。

25. Linux查看计划任务的命令

crontab -l（当前用户）、cat /etc/crontab、ls /etc/cron.*、cat /var/spool/cron/用户名。

26. PHP反序列化的原因

unserialize() 处理用户可控数据时，自动调用对象的魔术方法（__wakeup、__destruct等），攻击者构造 POP 链执行恶意操作。

27. 看告警怎么判断Webshell上传是否成功，如果判断出来上传成功怎么去处理？

判断：响应状态码200/201，返回文件路径或访问时能解析。处理：立即隔离主机，备份Webshell及日志，分析上传漏洞点并修复，全盘扫描查杀，清除后修改口令。

28. Python爬虫常用的库

requests、beautifulsoup4、lxml、scrapy、selenium、playwright、aiohttp、re。

  • requests：发送 HTTP/HTTPS 请求，简化网络操作，适合普通网页抓取。
  • beautifulsoup4：解析 HTML/XML，提供导航、搜索和修改文档树的功能，常配合 requests 使用。
  • lxml：解析 HTML/XML，支持 XPath，速度快，与 beautifulsoup4 解析器互配合。
  • scrapy：专业的爬虫框架，支持异步、中间件、管道，适合大规模爬取。
  • selenium：自动化浏览器操作，可模拟点击、滚动，适合需要 JS 渲染的动态页面。
  • playwright：现代化浏览器自动化工具，支持多浏览器，速度更快，可拦截网络请求。
  • aiohttp：异步 HTTP 客户端/服务器库，适合高性能并发爬取。
  • re：Python 内置正则表达式模块，用于从文本中提取、替换符合规则的字符串。

29. 拿到一个登录框除了暴力破解、万能密码还能做什么？

测试SQL注入（联合/盲注）、XSS、用户名枚举、逻辑漏洞（验证码绕过、密码重置）、框架漏洞（Shiro/Struts2）、CSRF、会话固定、默认口令。

31、Qzzty是怎么做信息收集的？

首先我认为攻防，渗透的本质就是信息收集，做信息收集的主要目的，是获取更多的资产，收集别人 没有找到的资产，利用信息不对称来建立优势，我认为这是非常重要的。

我把信息收集分成被动信息收集和主动信息收集。

被动信息收集是不触碰到目标的，无感知的信息收集。

一是通过icp备案，企业查询来获取目标单位的相关信息。企业查询我通常使用天眼查，企查查，爱企查，风鸟，小蓝本。其中风鸟和小蓝本是免费的，小蓝本还可以查询到新媒体，像公众号，小程序这种，也很有帮助。

通过icp备案来拿到域名和企业名称，在企业查询中，会拿到企业名称，域名，邮箱，注册人。

通过域名来进行子域名的收集，IP段的收集，在这里要进行一件事就是对cdn进行一个识别判断，找到目标真实ip。这里，真实ip的c段也列为可能目标。

邮箱有时候可以获取域名信息，还可以通过发邮件到邮件服务器来识别目标网站的真实ip，另外也可以用来钓鱼。

注册人的信息，主要是进行一个whois反查，拿到更多域名或者其他信息。也可以钓鱼。

通过企业名称来查询旗下控股信息超过50%的子公司，把子公司的信息收集出来，再进行新一轮同上的信息收集。

二是空间测绘引擎，fofa,hunter,quake我主要使用这三个，交叉使用，还有钟馗之眼等，用的比较少。空间测绘主要是通过语法来收集对应信息，比如icp备案，domian域名，ip段,同一系统的title，或者js中的特征来找。还有一个很重要的就是图标hash来进行一个测绘。

三是新媒体收集，小程序，app，公众号，服务号，可能拿到额外的域名和ip，或者一些有用的社工信息。

四是一些偏门的收集

(1)是利用网站的证书去收集到其他关联资产

(2)是通过谷歌语法来收集信息，可能会找到一些泄露的敏感文件，目录，或者社工信息。
(3)是一些代码托管平台，比如github，去找找可能存在相关源码的泄露

(4)一些公开网盘，比如百度网盘，语雀知识库，凌云风网盘等等

(5)是隐晦的信息收集，国外的社工库，飞机电报

接下来是主动信息收集

通过被动信息拿到的很多的域名，子域名，IP，进行一个全端口扫描加目录遍历，最后分为web应用和其他服务。web应用作为重点测试对象，其他服务通过口令爆破，端口服务的一些经典nday去利用。

子域名爆破常用的工具有subfinder,oneforall，子域名挖掘机。

还有用ARL灯塔进行主动的信息收集。

端口扫描常用的nmap,masscan，rustscan,nabbu，或者使用无影。

目录扫描使用的工具是dirsearch,gobuster,ffuf,这里会使用一个目录递归扫描。

然后使用httpx，或者无影，对web资产进行指纹识别，梳理存活且可能高价值的资产，比如OA，后台管理系统等等。

对于403的资产，也会保存下来，尝试一下403bypass，如果成功，升级为重点资产。

32、Qzzty如何处理子域名爆破的泛解析问题

泛解析（通配符DNS解析），就是在域名DNS记录中配置 * 通配规则，所有没有单独设置解析记录的随机子域名，都会被统一解析到同一个IP地址。举个例子，域名test.com开启泛解析后，不管是aaa.test.com、123.test.com这类不存在的子域名，都能正常解析出IP；它常被用来兼容临时子域名、阻挡简单的子域名爆破，但也会导致爆破工具出现大量无效假结果，因此专业子域名工具都会自带泛解析识别与过滤功能。

(1) 泛解析的探测与识别

在进行子域名爆破之前，第一步是确定目标域名是否开启了泛解析

操作方法：

  • 随机生成一个不存在的子域名：例如，random-string-123.example.com。这个字符串要足够随机且复杂，以确保它不可能是真实存在的子域名

  • 对其进行 DNS 查询：使用 ping、dig 或 nslookup 等命令来查询其 IP 地址

  • 记录返回的 IP 地址：如果返回了一个 IP 地址，那么这个地址很可能就是泛解析的地址

为了提高准确性，你可以多生成几个随机子域名并重复上述步骤。如果它们都解析到相同的 IP 地址，那么就可以确认泛解析已开启，且你已经找到了泛解析的 IP 地址

(2) 爆破过程中的泛解析过滤

确认泛解析后，在进行子域名爆破时，你需要过滤掉所有解析到泛解析 IP 地址的结果.

现在很多子域名爆破工具都内置了泛解析过滤，比如subfinder等。

33、Qzzty如何绕过 CDN 查找真实 IP

CDN 全称内容分发网络，简单讲就是一套遍布各地的边缘缓存服务器集群。它会提前把网站页面、图片、脚本等资源同步到就近节点，用户访问时不再直连网站源服务器，而是从附近节点获取内容，既能加快访问速度、减轻源站压力，还能隐藏服务器真实 IP、分流流量并抵御部分攻击，这也是需要专门方法才能绕过它找到源站 IP 的原因

（1）首先我会通过多地ping，全球ping来看是否返回了不同的ip地址，如果返回了很多ip，则目标开启了cdn。如果返回了很多ip，其中有一个ip出现的次数很多，可以猜测为真实ip。使用全球ping去判断，有些cdn会解析国内不同节点，但是不解析国外，通过全球Ping返回的ip地址，也可能是真实的ip地址。

（2）通过子域名判断，有的主域名部署cdn，但是子域名没有cdn，这些子域名可能与主域名部署在同一个服务器上，或者在同一个 IP 段内。把大量的子域名进行一个ip的查询，多次返回稳定的结果，推测为真实ip。

（3）邮件头信息
● 原理: 很多网站的邮件服务器（如 SMTP）与 Web 服务器部署在同一台机器上，而邮件服务器通常不使用 CDN
● 方法:
○ 尝试给目标网站的邮箱（如 admin@example.com）发送一封邮件
○ 在你的邮箱中查看这封邮件的原始邮件头（Original Headers）
○ 在邮件头中寻找 Received: from 字段。这个字段记录了邮件在传输过程中经过的服务器IP 地址。通常，第一个 Received: from 后面的 IP 地址就是邮件服务器的真实 IP，而这个 IP 很可能就是网站的真实 IP

（4）历史 DNS 记录查询
● 原理: 一个网站在上线 CDN 之前，其 DNS A 记录直接指向的就是真实 IP。一些服务会保存这些历史记录
● 方法:
○ 使用在线服务（如 SecurityTrails、WhoisXML API 或 ViewDNS）查询域名的历史 DNS 解析记录
○ 通过这些历史记录，你可以找到在 CDN 启用之前网站使用的真实 IP 地址

（5）SSL 证书查询
● 原理: SSL 证书可能会包含一些指向源站的信息，例如在某些情况下，证书颁发者会记录申请者的 IP 地址
● 方法:
○ 通过在线服务（如 Censys、Shodan）对目标域名进行 SSL 证书查询
○ 查看证书的 subjectAltName 字段，或者检查是否有其他关联信息泄露了真实 IP
○ 在某些情况下，如果目标网站使用了与源站 IP 绑定的证书，可以在 Shodan 等搜索引擎中直接搜索证书哈希或序列号来定位真实 IP

34、如果给Qzzty一场攻防演练，Qzzty会怎么打

第一件事是解读规则，首先是看资产范围界定，是否严格，如果限制到ip+端口这种很严格的，非xxx域名不收取的，那么就先不找ipc段和额外的子域名收集，先对给定的资产进行测试。如果规则比较宽松，那么在对给定资产进行测试的同时，会花更多的时间在信息收集上，c段，子域名，新媒体，偏远资产等。第二看漏洞收取范围标准，如果说对于无法利用或者利用困难，意义不大的漏洞，无敏感信息泄露的未授权，错误配置页面暴露，甚至是swgger泄露等标注了不收取，那么就不在这些上面浪费时间。还有包括重复是否加分等等，如果重复不算，那么就要先打简单的，追求速度。第三看额外加分情况，比如组合利用等等。

第二件事就是根据规则来指定扫描策略。资产梳理完毕后，优先进行top100,top1000的扫描，抓取其中的脆弱资产进行测试。如果目标单位防护很严格，成功很小，考虑进行全端口扫描，细致排查。

举例我在某省公安厅参加的一次视频网的专项渗透，给的资产非常多，top100扫描都要花大量的时间，我上来先找几个/24小网段初步扫描，发现大部分的视频设备资产都部署在80端口，于是我就直接针对这一个80端口进行大范围扫描，但是这个范围也很多，追求速度，用精简指纹去检测，不匹配poc，同时对web界面进行一个截图，最后来判断资产分布。把相同的视频设备提取出来，进行poc扫描，然后很快命中了一些历史rce,xml注入等nday，利用其中一个nday（海康威视的2017）拿到了密码，进行密码喷洒，发现某个/16网段的所有设备都是该密码。发现某个ip不是视频系统，是xx业务系统，推测它可能是业务主要ip，提取出来进行全端口的扫描。

总结就是，我会根据演练的规则，针对性制定不同策略，以此来完成渗透测试。

35、Qzzty讲讲xss漏洞的原理，利用，防御，反射型，存储型，DOM型xss的核心区别

XSS 全称跨站脚本攻击，原理是Web 应用未对用户可控的输入内容做过滤、转义、编码处理，导致攻击者可以注入恶意脚本【主流为 JavaScript，也可搭配 HTML 事件、VBScript（微软的老旧语言，少见了） 等】。

当其他用户访问受影响页面时，受害者的浏览器会将注入的恶意内容当作正常页面代码解析并执行，最终达成攻击目的。漏洞运行在客户端，用户的浏览器。

反射型xss，恶意脚本不会存储在服务器，而是被浏览器直接执行一次，比如页面搜索框。

存储型xss，恶意脚本会被保存到服务器，所有访问该页面的用户都会触发攻击，比如留言板，评论区，个人简介等等。

DOM型xss，恶意代码是不经过服务器的，是由前端的js直接解析导致的。

危害和利用：

cookie和session窃取，页面篡改，植入黑链和广告，钓鱼欺骗攻击，实时监听输入，比如键盘记录，xss+csrf组合攻击，内网探测

如何防御

  • 进行白名单的校验，只允许安全的协议：只接受 http 和 https 协议。所有其他协议，特别是 javascript:、data:、vbscript: 等，都应该被拒绝

  • 进行实体化编码，把恶意代码转化成纯文本，让浏览器无法解析，核心字符就是<>，单引号，双引号。

  • Cookie 设置 HttpOnly， 禁止 JavaScript 读取用户的 Cookie

  • 禁用一些危险函数：innerHTML、document.write、eval()、setTimeout(字符串)

  • 设置csp安全策略

csp全程内容安全策略，是浏览器自带的安全机制，作用是告诉浏览器是否只允许加载执行信任的资源，对未知，恶意的脚本进行拦截。可以设置直接拦截模式和日志记录（不拦截）两种模式。
