三、知识点

1、什么是 rbash？

rbash 是 Bash 的一种受限运行模式，全称 Restricted Bash。
在 Linux 系统中，可以通过以下几种方式进入 rbash：

  • 直接执行 rbash 命令（通常是 /bin/rbash，它实际上是一个指向 /bin/bash 的硬链接或拷贝，bash 检测到自己的名字是 rbash 后就会进入受限模式）。
  • 使用 bash -r 或 bash --restricted 启动。
  • 将用户的登录 shell 设置为 /bin/rbash（例如在 /etc/passwd 中），这样用户登录后自动进入受限环境。

核心思想是：为不可信用户提供一个功能受限的 Shell，限制其对系统的操作能力，但又允许执行少量必需的命令。

2、rbash 的具体限制

当 Bash 以受限模式运行时，以下行为被禁止：

限制类型 具体表现
目录跳转 禁止使用 cd 命令改变工作目录
环境变量 不能修改 SHELL、PATH、ENV、BASH_ENV 等关键变量（但可以通过 export 新增或覆盖其他变量）
命令路径 执行的命令 **不能包含斜杠 /**，即不能指定绝对路径或相对路径来运行命令（如 /bin/sh、./script）
重定向 禁止使用 >、>>、<、<>、>& 等重定向操作符
内置命令限制 禁止使用 enable 来启用或禁用内置命令；禁止用 command -p 运行命令
SHELLOPTS / BASHOPTS 不能修改这些只读变量
函数定义 允许定义函数，但函数体内不允许包含重定向等受限操作
退出方式 不能用 exec 替换当前 shell，但可以用 exit 退出
文件描述符 禁止显式移动或关闭文件描述符（如 exec >&2）

此外，管理员通常还会配合其它手段增强限制：

  • 设置一个极简的 PATH（如 /home/user/bin 且该目录下只有少数几个安全命令）。
  • 将用户的家目录权限设为不可写，防止用户添加恶意文件。
  • 移除或限制用户对某些命令的访问（通过文件权限）。

3、rbash 的常见用途

  • 蜜罐/CTF 靶机：模拟一个受限环境，要求参与者绕过限制进行提权。
  • 受限远程访问：给外部人员一个仅能执行特定命令的 Shell，比如只允许查看日志、重启服务等。
  • 跳板机/堡垒机：在用户登录后限制其操作范围，防止误操作或恶意行为。
  • 应用容器：某些 Docker 容器或 Jail 环境内使用 rbash 作为初始 Shell。

本质上，它是一种 安全边界，但 rbash 本身并不是一个完整的安全解决方案，因为其设计存在许多绕过方法。

4、rbash 逃逸的基本思想

rbash 的限制主要集中在：

  1. 不能运行带 / 的程序 → 必须通过系统路径中的命令运行；
  2. 不能改变目录 → 无法访问其他目录下的工具；
  3. 不能重定向输出 → 限制了数据写出；
  4. 不能修改 SHELL 和 PATH 等环境变量 → 维持受限环境。

逃逸的核心思路是：利用受限环境中仍然可用的命令或功能，打破上述某一项或多项限制，最终获得一个不受限的 Shell。
常见突破口包括：

  • 利用允许的命令内置的“执行外部命令”功能（如 vi、man、find、less 等）。
  • 利用 Shell 自身的特性（如变量、函数、数组）注入或重新定义命令。
  • 利用编程语言解释器（Python、Perl、Ruby 等）直接调用系统调用。
  • 利用环境变量传递中的逻辑漏洞（如 BASH_CMDS 哈希表污染）。
  • 如果系统存在配置错误（如 SUID 文件、sudo 配置不当），则可以借助它们直接提权到更高权限，从而绕过 rbash。

7、如何防御 rbash 逃逸？

管理员如果希望 rbash 变得可靠，需要采取纵深防御策略：

  • 严格控制 PATH：将 PATH 设为一个只包含绝对安全程序的目录，不包含 vi、man、find、awk、python 等可用于逃逸的程序。
  • 移除或禁用危险命令：直接删除或限制用户对这些二进制文件的执行权限（chmod 700 目录等）。
  • 使用受限文件系统：采用 chroot、jail 或 Linux 命名空间/容器技术，将用户的文件系统可见范围缩小到最小。
  • 禁止 Shell 解释器：如果不需要脚本语言，就不要安装 Python/Perl 等。
  • 限制 Bash 版本并打补丁：使用最新版 Bash，确保已知逃逸漏洞被修复。
  • 配合 seccomp 或 AppArmor/SELinux：通过强制访问控制进一步限制进程能执行的系统调用和文件。
  • 不以 rbash 作为唯一防线：rbash 本质是“软限制”，只能防君子不能防高手，真正的安全需要靠权限隔离、强制访问控制和监控。

8、总结

  • rbash 通过禁用目录跳转、禁止命令含斜杠、禁止重定向、锁定环境变量等手段来限制用户操作。
  • rbash 逃逸的核心是利用环境中残留的、能够执行外部命令的工具（编辑器、分页器、查找工具、脚本语言）或者利用 Bash 内部数据结构（如 BASH_CMDS、函数定义）绕过限制。
  • 熟知这些方法，无论是对于 CTF 玩家、安全测试人员，还是系统管理员加固系统，都具有重要意义。
  • 防御时必须将 rbash 置于多层安全措施之后，切不可将其视为坚不可摧的屏障。
