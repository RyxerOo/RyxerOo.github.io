curl 发送 HTTP 请求参数完全指南

curl 是一个功能强大的命令行工具，支持超过 200 种选项。本文档聚焦于发送 HTTP/HTTPS 请求时最常用、最核心的参数，并按照功能分类，辅以丰富的示例，帮助你快速查阅和运用。

1. 请求方法与数据发送

参数 说明 示例
-X, --request <method> 指定 HTTP 方法（默认 GET） -X POST
-d, --data <data> 发送数据到请求体（自动转为 POST） -d "name=John&age=30"
--data-ascii <data> 以 ASCII 编码发送数据 同上
--data-binary <data> 发送二进制数据（保留原样） --data-binary @file.bin
--data-raw <data> 发送数据，不解析 @ 字符（适用于以 @ 开头的字符串） --data-raw "@rawstring"
--data-urlencode <data> 对数据进行 URL 编码后再发送 --data-urlencode "name=John Doe"
-F, --form <name=content> 模拟 multipart/form-data 表单提交，支持文件上传（@ 前缀） -F "file=@/path/to/file"
-G, --get 将 -d 的数据附加到 URL 查询字符串（用于 GET） -G -d "key=value"
-I, --head 仅获取响应头（HEAD 请求） -I https://example.com
-T, --upload-file <file> 通过 PUT 方法上传文件 -T localfile https://example.com/upload

注意：使用 -d 时未指定 -X，curl 默认发送 POST 请求。若同时使用 -G，则变为 GET。

2. 请求头与认证

参数 说明 示例
-H, --header <header> 自定义请求头（可多次使用） -H "Content-Type: application/json"
-A, --user-agent <name> 设置 User-Agent -A "Mozilla/5.0"
-e, --referer <URL> 设置 Referer -e "https://example.com"
`-b, –cookie <data file>` 发送 Cookie（字符串或文件）
-c, --cookie-jar <file> 将响应中的 Cookie 保存到文件 -c cookies.txt
-u, --user <user:password> HTTP 基本认证 -u admin:secret
-H "Authorization: ..." 手动设置 Bearer Token 等 -H "Authorization: Bearer TOKEN"
--anyauth 自动协商服务器支持的认证方法 与 -u 搭配使用
--digest 使用 HTTP 摘要认证（Digest） --digest -u user:pass
--ntlm 使用 NTLM 认证 --ntlm -u user:pass

3. 输出与重定向

参数 说明 示例
-o, --output <file> 将响应内容保存到指定文件 -o response.json
-O, --remote-name 从 URL 中提取文件名并保存 -O https://example.com/file.zip
-L, --location 跟随 HTTP 重定向（默认不跟随） -L https://short.url
--max-redirs <num> 限制最大重定向次数（配合 -L） --max-redirs 5
-D, --dump-header <file> 将响应头保存到文件 -D headers.txt
--resolve <host:port:ip> 手动指定域名解析（绕过 DNS） --resolve example.com:443:127.0.0.1

4. 调试与信息

参数 说明 示例
-v, --verbose 输出详细通信过程（请求/响应头、握手信息） -v https://api.example.com
-s, --silent 静默模式（不显示进度条和错误信息） -s https://example.com
-S, --show-error 与 -s 搭配，仅显示错误信息 -sS https://example.com
-i, --include 在输出中包含响应头（正文之前） -i https://example.com
--compressed 请求压缩（如 gzip），并自动解压响应 --compressed https://example.com
-w, --write-out <format> 定义输出格式，常用 %{http_code} 等 -w "HTTP %{http_code}\n"
--trace <file> 将完整的跟踪日志写入文件（更详细） --trace trace.log
--trace-ascii <file> 与 --trace 类似，但输出为 ASCII --trace-ascii trace.txt

5. SSL/TLS 与连接选项

参数 说明 示例
-k, --insecure 跳过 SSL 证书验证（⚠️ 仅测试环境） -k https://self-signed.badssl.com
--cert <file> 指定客户端证书文件（PEM 格式） --cert client.pem
--cert-type <type> 证书类型（PEM/DER/P12） --cert-type PEM
--key <file> 指定私钥文件 --key key.pem
--key-type <type> 私钥类型（PEM/DER） --key-type PEM
--pass <phrase> 私钥密码 --pass "mypass"
--cacert <file> 指定 CA 证书文件进行验证 --cacert ca-bundle.crt
--capath <dir> 指定 CA 证书目录 --capath /etc/ssl/certs/
--ciphers <list> 指定使用的加密套件 --ciphers ECDHE-RSA-AES128-GCM-SHA256
--tlsv1.2 强制使用 TLS 1.2 --tlsv1.2 https://example.com
--tlsv1.3 强制使用 TLS 1.3 --tlsv1.3 https://example.com
--connect-timeout <seconds> 连接超时时间（秒） --connect-timeout 10
-m, --max-time <seconds> 整个操作的最大允许时间（秒） -m 30

6. 代理与限速

参数 说明 示例
-x, --proxy <[protocol://]host[:port]> 使用代理（支持 HTTP/HTTPS/SOCKS） -x http://proxy.example.com:8080
--proxy-user <user:pass> 代理认证凭据 --proxy-user user:pass
--noproxy <host-list> 绕过代理的主机列表（逗号分隔） --noproxy localhost,127.0.0.1
--socks5 <host:port> 使用 SOCKS5 代理（快捷方式） --socks5 127.0.0.1:1080
--limit-rate <speed> 限制传输速率（单位：K/M/G） --limit-rate 100K
-y, --speed-time <seconds> 若低于最低速度，经过该时间后终止 -y 10 --speed-limit 1
-Y, --speed-limit <speed> 最低传输速度阈值 -Y 1（表示 1 字节/秒）

8. 最佳实践与注意事项

  • 安全第一：生产环境中避免使用 -k（跳过证书验证），应正确配置 CA 证书。
  • 避免将敏感信息暴露：命令中的密码、Token 等可能被 shell 历史记录捕获，建议使用交互式输入（如 -u 不加密码，会提示输入）或从文件读取。
  • 使用 --data-raw 避免 @ 解析：当你的数据恰好以 @ 开头时，用 --data-raw 防止 curl 误认为文件路径。
  • **复杂请求使用 --libcurl**：如果你需要将 curl 命令转换为编程语言的代码，可以使用 --libcurl <file> 生成 C 代码框架。
  • 网络超时设置：在高延迟环境中，务必设置 --connect-timeout 和 -m，防止进程卡死。
  • 批量请求：可以利用 xargs 或 parallel 与 curl 结合，实现并发请求，但注意控制并发数。
  • 内容类型匹配：发送 JSON 时务必设置 Content-Type: application/json，发送表单时默认 application/x-www-form-urlencoded，用 -F 则是 multipart/form-data。
